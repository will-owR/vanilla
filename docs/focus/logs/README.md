# E2E Smoke Logs

This directory contains reproducibility logs and artifacts generated by the lightweight Puppeteer e2e smoke script and related test helpers.

How to run the Puppeteer smoke test locally

1. Ensure both the client dev server and backend server are running:

```bash
# From repo root (example)
cd client && npm run dev &
cd ../server && npm run dev &
```

2. Run the Puppeteer e2e smoke script (requires Chrome and Node 18+):

```bash
node server/scripts/e2e-smoke.js
```

If your backend is protected in dev by `DEV_AUTH_TOKEN`, start the server with the same token in the environment so the Vite dev proxy and direct API fallback calls succeed. Example:

```bash
export DEV_AUTH_TOKEN=yourtoken123
# start server and client in separate terminals with this env var set
node server/index.js
cd client && npm run dev
# in the same environment run the smoke script
node server/scripts/e2e-smoke.js
```

Where logs are saved

- The smoke script saves manual run output and the team saves artifacts to `docs/focus/logs/` as timestamped `.json` files. Look for `e2e-smoke-<timestamp>.json` files.

Notes

- The script prefers a UI path (Generate → Preview) but falls back to direct API calls (POST /prompt -> GET /preview) if the UI path does not render a preview within the configured window.
- The script will include the `x-dev-auth` header on direct API calls when `DEV_AUTH_TOKEN` is present in the environment to avoid 401s.
